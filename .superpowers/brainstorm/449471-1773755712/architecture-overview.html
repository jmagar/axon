Key Design Decisions

Pulse Chat → SSE (not WebSocket)
The Android app calls POST /api/pulse/chat which returns Server-Sent Events. This is simpler than maintaining a persistent WebSocket — no reconnect logic, no multiplexing. OkHttp handles SSE natively.
Dashboard → REST polling
Stats/health/jobs fetched via REST on screen entry + pull-to-refresh. No persistent connection needed for a "quick glance" screen.
Room for conversation history
Conversations + messages stored locally in SQLite via Room. Works offline for reviewing past chats. No server-side conversation persistence needed.
No Hilt/Dagger DI
Manual dependency injection via a simple AppContainer object. Hilt adds complexity and magic that's hard to debug as a beginner. We can add it later if needed.
