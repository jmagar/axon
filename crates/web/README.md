# crates/web
Last Modified: 2026-02-25

Axum-based web runtime for `axon serve` (legacy static UI + websocket execution bridge).

## Purpose
- Serve a browser UI over HTTP.
- Bridge UI actions to CLI execution through websocket messages.
- Stream command output, logs, and Docker stats to connected clients.

## Responsibilities
- HTTP routes for static assets, output files, and download endpoints.
- Websocket connection handling and message dispatch (`execute`, `cancel`, `read_file`).
- Runtime broadcast of Docker stats and command execution events.
- Packaging/downloading generated crawl artifacts.

## Key Files
- `../web.rs`: server startup, route graph, websocket orchestration, and output file serving.
- `execute/mod.rs`: command execution handlers for websocket requests.
- `execute/events.rs`: websocket event framing.
- `execute/files.rs`: output directory and file helper logic.
- `download.rs`: artifact download endpoints.
- `docker_stats.rs`: Docker telemetry polling loop.
- `pack.rs`: output packaging helpers.
- `static/*`: legacy static UI assets.

## Integration Points
- Uses CLI execution paths by spawning/streaming command runs through execute handlers.
- Consumes output generated by crawl/jobs/vector modules and exposes artifacts to UI clients.
- Coexists with modern Next.js app in `apps/web`; this module remains the `axon serve` runtime.

## Notes
- In debug builds, static assets are read from disk for hot reload; release builds compile assets into the binary.
- Path traversal protections for output file serving are enforced via canonicalization and prefix checks.
