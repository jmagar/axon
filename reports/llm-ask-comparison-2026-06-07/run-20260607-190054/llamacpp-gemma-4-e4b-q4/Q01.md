  Assistant:
Bun does not run arbitrary lifecycle scripts from installed dependencies by default because doing so is considered a potential security risk, as these scripts could execute arbitrary code on your system [S2][S3][S15]. This security feature means that by default, Bun only runs scripts for packages that are already trusted [S2][S3][S15].

To allow a specific dependency's lifecycle scripts to run, the project must explicitly trust that package by adding its name to the `trustedDependencies` field in the project's `package.json` file [S2][S3][S15]. After adding the package to this list, the user must re-run the installation command for Bun to recognize and execute the lifecycle scripts for that specific package [S2][S3]. Alternatively, users can use `bun add --trust [package]` to both add the package and update the `trustedDependencies` list in `package.json` [S16].
