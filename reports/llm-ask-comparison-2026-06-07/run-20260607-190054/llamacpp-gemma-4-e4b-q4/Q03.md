  Assistant:
Subnet routers and exit nodes are both features that route traffic through a specific device on a Tailscale network, but they solve fundamentally different networking problems [S8, S9].

**Practical Difference:**

*   **Subnet Routers:** These are used to extend the Tailscale network (tailnet) to reach devices or entire networks that cannot directly run the Tailscale client, such as printers or entire Virtual Private Clouds (VPCs) [S1, S13]. They function as gateways, allowing tailnet devices to access non-Tailscale devices within specific, private subnets—this is used for accessing private LANs or cloud VPCs [S1, S12].
*   **Exit Nodes:** These devices are used to route *all* the outbound internet traffic from the tailnet devices through the exit node itself [S12, S8]. This is useful for accessing geo-restricted content or improving privacy, as your internet traffic appears to originate from the exit node's location [S1].

**SNAT Changes with Subnet Routers:**

By default, when subnet routers are used, Tailscale employs Source Network Address Translation (SNAT) (also called masquerading) [S1, S5]. When SNAT is active, traffic originating from a device behind the subnet router appears to have the subnet router's IP address as its source, rather than the original device's source IP address [S1, S5]. If preserving the original source IP address is necessary, SNAT can be disabled, which is done using a command like `tailscale up --snat-subnet-routes=false` on Linux machines [S5, S10].
