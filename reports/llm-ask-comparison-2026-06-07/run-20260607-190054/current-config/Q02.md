  Assistant:
HOT updates are possible when the update does not modify any columns referenced by the table’s indexes, except that current PostgreSQL excludes summarizing indexes from that restriction, and when there is enough free space on the same heap page that contains the old row to store the updated row version [S4].

When those conditions are met, HOT reduces two update overheads: it avoids creating new index entries for updated rows, though summary indexes may still need updates, and it allows obsolete intermediate row versions to be removed during normal operation, including `SELECT`s, rather than waiting for periodic vacuum operations [S4].
