  Assistant:
Bun does not run arbitrary lifecycle scripts such as `postinstall` for installed dependencies by default because those scripts can execute arbitrary code on your machine, making them a potential security risk and a possible vehicle for malicious scripts.[S2][S3][S4]

To allow a specific dependency’s lifecycle scripts to run, the project must add that package name to the `trustedDependencies` array in its `package.json`, then reinstall so Bun reads the field and runs that package’s lifecycle scripts.[S2][S3]

```json
{
  "trustedDependencies": ["my-trusted-package"]
}
```
