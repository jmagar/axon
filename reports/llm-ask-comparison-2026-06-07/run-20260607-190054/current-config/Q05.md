  Assistant:
A pool must already have at least one normal vdev, meaning a vdev that is not a **dedup** or **special** vdev, before devices can be assigned to the special allocation class. [S1]

Small-block placement is opt-in and is enabled per dataset by setting the `special_small_blocks` property to a nonzero value after a special-class vdev has been added to the pool. [S1][S7] This property sets the threshold: blocks smaller than or equal to the configured size are assigned to the special allocation class, while larger blocks remain in the regular class. [S7] The default is `0`, which means no small file or zvol blocks are allocated in the special class. [S7]

The special device’s redundancy should match the redundancy of the pool’s normal devices because special vdevs store internal metadata and optionally small file blocks, and ZFS documentation explicitly recommends matching their redundancy to the other normal devices in the pool. [S1] This matters because a faulted pool can result from corrupted metadata or from faulted devices with insufficient replicas to continue functioning. [S1]

If the special class fills up, allocations intended for it spill back into the normal class. [S1]
