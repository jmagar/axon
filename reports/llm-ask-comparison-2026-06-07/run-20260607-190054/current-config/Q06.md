In MCP sampling, the server sends `sampling/createMessage` through the client because MCP is designed for servers to request LLM completions via clients, letting the client control model access, model selection, and permissions while allowing the server to use AI capabilities without needing server-side API keys.[S11] This also supports trust-and-safety controls: the client-side application should keep a human in the loop, allow review or editing of prompts, and present generated responses for review before returning them.[S11]

Model selection is intentionally abstract because servers and clients may use different AI providers, the client may not have access to the exact model named by the server, or the client may prefer an equivalent model from another provider.[S7] Instead of requiring a precise model, MCP combines optional model hints with normalized priorities: `costPriority` favors cheaper models, `speedPriority` favors lower latency, and `intelligencePriority` favors more capable models when their values are higher.[S7] Hints can suggest specific models or model families, but clients make the final model selection based on the preferences and their available models, and the preferences may be ignored.[S4][S8]
