**Question:** Tailscale subnet routers and exit nodes both route traffic, but they solve different network problems. Explain the practical difference between them, including what SNAT changes about traffic from devices behind a subnet router.

  Assistant:
Subnet routers are for reaching **specific private subnets** from your tailnet, especially networks or devices that can’t run the Tailscale client, such as printers, cameras, legacy LANs, or cloud VPC resources. [S1] They act as gateways between the tailnet and conventional subnet-based networks, advertise routes for those private IP ranges, and let tailnet devices access non-Tailscale devices behind the router without changing general internet routing. [S1]

Exit nodes are for routing **outbound internet or non-Tailscale traffic** from tailnet devices through a chosen tailnet device, effectively making that device the egress point for internet traffic. [S1][S8] In practical terms, use a subnet router when you want “my laptop on Tailscale can reach 192.168.1.50 on the office LAN,” and use an exit node when you want “my laptop’s internet traffic should leave through this other device/location.” [S1][S8]

SNAT changes how traffic from devices behind a subnet router appears to the rest of the network: by default, Tailscale rewrites the source IP so traffic from a device behind the subnet router appears to come from the subnet router itself rather than from the original device. [S1][S5] This makes communication more transparent and reduces routing complexity, but it hides the original source IP and can prevent traversing multiple networks. [S5][S10] If SNAT is disabled with `--snat-subnet-routes=false` on Linux, the original source IP is preserved, but you must manually configure return routing so devices know to send Tailscale-destined traffic, such as `100.64.0.0/10`, back through the subnet router. [S1][S5]
