Bun's package manager does not run arbitrary lifecycle scripts for installed dependencies by default because executing such scripts poses a potential security risk, as they could run arbitrary code on the machine [S2][S3][S15]. While some popular packages have been added to a default allowlist in later versions, this protection is a core security feature of Bun [S15].

To explicitly allow a specific dependency's lifecycle scripts to run, a project must add the package name to the `trustedDependencies` array within its `package.json` file [S2][S3]. After updating `package.json` with this allow-list, the user must re-run the installation command for Bun to recognize the trusted package and execute its lifecycle scripts [S2][S3]. Additionally, one can use the `bun add --trust [package]` command to achieve this, which updates both the lockfile and the `package.json` [S16].
