  Assistant:
Subnet routers and exit nodes both manage traffic routing, but they serve fundamentally different purposes within the Tailscale network [S9]. A **subnet router** is designed to extend the tailnet's reach to non-Tailscale devices or entire private networks, allowing tailnet devices to access specific, private subnets that do not run the Tailscale client [S1, S6, S11]. Conversely, an **exit node** routes *all* the device's outbound internet traffic through itself, making the exit node's location the apparent source of all internet-bound traffic [S12, S18].

The practical difference is defined by the destination and scope of the traffic:
*   **Subnet Router:** Provides access to internal, private IP ranges (like an office LAN or AWS VPC) by acting as a gateway; it does not affect the device's general internet traffic routing [S1, S12].
*   **Exit Node:** Changes the *entire* internet exit path for the connected device; this is useful for masking the source location for general internet access [S12].

Regarding SNAT, subnet routers use Source Network Address Translation (SNAT) by default [S1]. When SNAT is enabled, traffic originating from a device located behind the subnet router will appear to have originated from the subnet router itself, rather than the actual original device's IP address [S1, S5]. If preserving the original source IP address is necessary, SNAT can be disabled, though this requires manual routing configuration [S1, S5].
