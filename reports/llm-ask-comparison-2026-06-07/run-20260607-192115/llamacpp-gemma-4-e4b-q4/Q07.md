The components that constitute the block hash in vLLM's prefix caching system include the parent hash value, the tokens contained within the current block, and extra hashes [S1]. The inclusion of the exact block tokens is necessary to minimize the possibility of hash value collision [S1]. Furthermore, to ensure uniqueness, other values such as LoRA IDs, multi-modality input hashes, and cache salts (used for isolation in multi-tenant environments) are incorporated into the hash calculation [S1].

The design specifies that only **full blocks** are cached [S1][S2][S5].

Regarding changes to the hash collision risk, in previous versions, the hash key was not guaranteed to be collision-free [S1][S5]. As of v0.11, the default hashing algorithm has been updated to `sha256`, which is implemented to address potential collision risks [S1][S5]. For achieving a deterministic, reproducible, and cross-language compatible hash, using `sha256_cbor` is recommended, though `xxhash_cbor` is also available for this purpose [S1][S5].

The components that constitute the block hash in vLLM's prefix caching system include the parent hash value, the tokens contained within the current block, and extra hashes [S1]. The inclusion of the exact block tokens is necessary to minimize the possibility of hash value collision [S1]. Furthermore, to ensure uniqueness, other values such as LoRA IDs, multi-modality input hashes, and cache salts (used for isolation in multi-tenant environments) are incorporated into the hash calculation [S1].

The design specifies that only **full blocks** are cached [S1][S2][S3].

Regarding changes to the hash collision risk, in previous versions, the hash key was not guaranteed to be collision-free [S1][S3]. As of v0.11, the default hashing algorithm has been updated to `sha256`, which is implemented to address potential collision risks [S1][S3]. For achieving a deterministic, reproducible, and cross-language compatible hash, using `sha256_cbor` is recommended, though `xxhash_cbor` is also available for this purpose [S1][S3].
