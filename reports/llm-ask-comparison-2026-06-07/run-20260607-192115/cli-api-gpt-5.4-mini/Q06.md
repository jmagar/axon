  You: In MCP sampling, why does the server send a `sampling/createMessage` request through the client instead of directly calling an LLM provider, and how do model hints and priority values affect model selection without guaranteeing an exact model?
  Assistant:
MCP has the server send `sampling/createMessage` to the **client** because the protocol is designed so the client, not the server, keeps control over model access, model selection, and permissions, can enforce safety/quotas, and can do so without the server needing LLM API keys. [S11][S1] The flow also supports a human-in-the-loop review step, where the client can review the request and response before delivering it. [S11][S3][S5]

Model selection in MCP is preference-based rather than an exact model command: servers provide normalized priorities for `costPriority`, `speedPriority`, and `intelligencePriority`, and may add `hints` that suggest a model name or family. [S2][S4][S5][S7] The hints are only suggestions, can be partial, can be evaluated in preference order, and clients may map them to equivalent models from different providers. [S4][S5][S7] The client makes the final model choice from its available models and may ignore the preferences entirely, so a hint like `claude-3-sonnet` does not guarantee that exact model will be used. [S4][S8][S2]
