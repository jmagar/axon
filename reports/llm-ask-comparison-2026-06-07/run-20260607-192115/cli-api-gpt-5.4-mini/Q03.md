  Assistant:
Subnet routers and exit nodes both forward traffic, but they solve different problems: an exit node routes outbound internet traffic from tailnet devices, while a subnet router provides access to specific private subnets and non-Tailscale devices without affecting general internet traffic routing. In practice, use an exit node when you want all or most internet-bound traffic to exit from a chosen Tailscale device, and use a subnet router when you want tailnet devices to reach an office LAN, cloud VPC, printer, or other private network resource. [S1][S6][S8]

For devices behind a subnet router, SNAT changes how their traffic appears to the rest of the network. By default, traffic from those devices is source-NATed so it appears to come from the subnet router itself rather than from the original host, which makes routing simpler. [S1][S5][S10]

If you disable SNAT, the original source IP is preserved in the packet, but then the devices on the far side need a return route for Tailscale traffic so replies can get back through the subnet router. The docs say you may need to configure that return path for the `100.64.0.0/10` Tailscale IP range on the device OS, in the VPC, or via DHCP. [S1][S5]
