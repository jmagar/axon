  Assistant:
In Bun's package manager, arbitrary lifecycle scripts (such as `postinstall`) do not run by default because they represent a potential security risk, as they can execute arbitrary or malicious code on your machine [S2][S3][S4]. 

To allow a specific dependency's lifecycle scripts to run, a project can use one of the following approaches:
* **Configure `package.json`:** Add the package name to the `trustedDependencies` array in the project's `package.json` file [S2][S3][S15]. After adding it, re-install the package so Bun can read this field and execute the scripts [S2].
* **Use CLI Install Flags:** Add the dependency using the command `bun add --trust [package]`, which automatically installs the package and adds it (along with its transitive dependencies) to the `trustedDependencies` list in the `package.json` [S16]. The `--trust` flag can also be used during updates [S5].
* **Use Bun PM commands:** Run the CLI command `bun pm trust [package]` to explicitly trust and run the scripts for a specific package, or run `bun pm trust --all` to trust and run scripts for all blocked dependencies [S13][S16].
