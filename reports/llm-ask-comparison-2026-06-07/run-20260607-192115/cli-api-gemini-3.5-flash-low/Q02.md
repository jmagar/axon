The heap-only tuple (HOT) optimization in PostgreSQL applies only when the following conditions are met during an update:
* The update does not modify any columns referenced by the table's indexes, which includes expression and partial indexes but excludes summarizing indexes such as BRIN [S1][S4].
* There is sufficient free space for the updated row on the page that contains the old row [S1][S4].

When these conditions are satisfied, HOT reduces overhead by providing two optimizations:
* It eliminates the need for new index entries to represent the updated rows, though summary indexes may still need to be updated [S1][S4].
* It allows old or intermediate versions of updated rows to be completely removed during normal operation (including during `SELECT` queries) instead of requiring periodic vacuum operations [S1][S4].
