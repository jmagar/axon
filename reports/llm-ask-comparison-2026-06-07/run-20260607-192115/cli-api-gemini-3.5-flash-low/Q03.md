  Assistant:
Exit nodes route outbound public internet traffic from your tailnet devices, functioning as VPN servers so that your internet activity appears to come from the exit node's location [S1][S13][S20]. This is practically useful for protecting traffic on untrusted networks or accessing geo-restricted online services [S1][S13][S20]. In contrast, subnet routers do not affect general internet traffic routing; instead, they act as gateways to specific private subnets, enabling tailnet devices to reach legacy or non-Tailscale devices within networks like office LANs or cloud VPCs [S1]. 

By default, Tailscale performs Source Network Address Translation (SNAT), also known as masquerading, on traffic passing through subnet routers [S1][S2]. When SNAT is active, traffic from a device behind a subnet router is modified so that it appears to originate from the subnet router itself rather than the original device [S1][S4][S11]. Disabling SNAT (which is supported on Linux subnet routers) preserves the original source IP address of the host [S1][S2][S11]. However, if SNAT is disabled, devices behind the subnet router do not automatically know how to route return traffic back to Tailscale IPs, requiring administrators to manually configure a return route on the devices, in the VPC settings, or via a DHCP server [S2][S4].
