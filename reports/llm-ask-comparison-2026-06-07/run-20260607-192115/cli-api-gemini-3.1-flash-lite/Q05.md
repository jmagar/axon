  Assistant:
To use OpenZFS special allocation classes, the storage pool must already contain at least one normal (non-special, non-dedup) virtual device [S1]. Once a special-class device has been added to the pool, small-block placement is enabled by setting the `special_small_blocks` property on a per-dataset basis to a nonzero value [S1][S9]. This property defines the threshold block size; blocks at or below this size will be assigned to the special allocation class, while larger blocks will be assigned to the normal class [S8][S15].

It is recommended that the redundancy of the special device match the redundancy of the other normal devices in the pool [S1][S2]. This is advised because special devices are dedicated to storing critical internal metadata, indirect blocks of user data, and (optionally) small file blocks; maintaining matching redundancy ensures that the special device provides the same level of fault tolerance as the rest of the pool [S1][S4].
