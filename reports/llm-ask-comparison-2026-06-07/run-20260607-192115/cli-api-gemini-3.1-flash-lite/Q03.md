Tailscale subnet routers and exit nodes serve different networking purposes within a tailnet, though both function as gateways for traffic [S1][S10].

*   **Subnet routers** are designed to provide access to specific private subnets (such as office LANs or cloud VPCs) [S1]. They allow devices in your tailnet to communicate with non-Tailscale devices—such as printers, cameras, or legacy servers—without installing the Tailscale client on those endpoints [S1][S12][S14]. They do not affect general internet traffic routing [S1].
*   **Exit nodes** function like VPN servers for your tailnet devices [S1][S9]. When you route traffic through an exit node, all of your internet traffic appears to originate from the exit node's location [S1][S20]. This is typically used to secure traffic on untrusted networks, such as public Wi-Fi, or to access geo-restricted content [S1][S13][S20].

### SNAT and Traffic from Subnet Routers
By default, subnet routers use Source Network Address Translation (SNAT), also known as masquerading [S1][S4]. When SNAT is active, traffic from a device behind the subnet router appears to the destination as if it originated from the subnet router itself rather than the original source device [S1][S11][S33]. 

While this default behavior reduces routing complexity, you can disable it on Linux subnet routers using the `--snat-subnet-routes=false` flag [S1][S4][S11]. Disabling SNAT preserves the original source IP address of the device behind the router [S1][S2]. However, if SNAT is disabled, devices behind the subnet router will not automatically know how to route return traffic back from the tailnet, requiring manual configuration of return routes on your DHCP server, VPC settings, or the device's operating system [S1][S2].
