In Bun's package manager, arbitrary lifecycle scripts (such as `postinstall`) are not executed for installed dependencies by default because they represent a potential security risk, as they can execute arbitrary code on a user's machine [S2][S3][S4].

To allow a specific dependency's lifecycle scripts to run, you must add that package to the `trustedDependencies` field in your project's `package.json` file [S2][S3][S4]. Once added, you can run an install command (such as `bun install`), and Bun will read the field and execute the lifecycle scripts for those trusted packages [S2][S3]. Alternatively, you can use the `bun add --trust [package]` command to add the dependency and its transitive dependencies to the `trustedDependencies` list automatically [S16].
