  Assistant:
HOT can be used when an update does not modify columns referenced by the table’s indexes and when there is enough free space on the same heap page as the old row to store the updated row version; current docs note that summarizing indexes are excluded from that indexed-column restriction, with BRIN identified as the core summarizing index method [S4].  

When those conditions are met, HOT reduces two main update overheads: it avoids creating new index entries for the updated row, though summary indexes may still need updates, and it allows obsolete row versions to be removed during normal operation, including `SELECT`s, instead of waiting for periodic vacuum work [S4].  

Older PostgreSQL 12 documentation states the indexed-column condition more broadly as “including expression and partial indexes” and describes the cleanup benefit as complete removal of old updated-row versions during normal operation rather than requiring vacuum, so the precise wording differs across documented versions [S1][S4].
