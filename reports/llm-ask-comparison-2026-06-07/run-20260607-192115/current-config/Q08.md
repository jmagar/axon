MVCC creates maintenance pressure because PostgreSQL updates add new row versions to the table rather than overwriting rows in place; those updates can also require new index entries for each updated row, and later removal of obsolete row versions and their index entries can be expensive [S1].  

HOT reduces part of that pressure when the update does not modify indexed columns, except summarizing indexes, and when there is enough free space on the page containing the old row for the new row version [S7]. In that case, PostgreSQL can avoid creating new index entries for the updated row, though summary indexes may still need updates [S7]. HOT can also let intermediate row versions be removed during normal operation, including `SELECT`s, rather than waiting for periodic vacuum, because indexes continue to refer to the original row version’s page item identifier, which can become a redirect into the HOT chain [S7].  

The visibility map addresses a different pressure point: indexes do not contain tuple visibility information, so a normal index scan must fetch the heap tuple for each matching index entry to check whether the current transaction can see it [S4]. Vacuum maintains the visibility map to record pages whose tuples are all known visible to active and future transactions until the page is modified [S4]. During an index-only scan, PostgreSQL checks the visibility-map bit for the heap page corresponding to a candidate index entry; if the bit is set, the row is known visible and the heap fetch can be skipped, but if it is not set, PostgreSQL must visit the heap tuple to check visibility [S3]. This can be much cheaper because the visibility map is vastly smaller than the heap and can often stay cached even when the heap is large [S4].  

Together: MVCC updates create extra heap versions and often extra index entries; HOT avoids some index churn and cleanup work for qualifying updates; and the visibility map lets index-only scans skip heap visibility checks on all-visible pages, especially for pages whose rows are relatively unchanging [S1][S7][S3].
