  Assistant:
A pool must already have at least one normal vdev—that is, a non-`dedup` and non-`special` vdev—before devices can be assigned to the special allocation class.[S1]

Small-block placement is opt-in per dataset: after a special-class vdev has been added, set the dataset’s `special_small_blocks` property to a nonzero threshold, and blocks smaller than or equal to that value are assigned to the special class while larger blocks remain in the regular class.[S1][S7] The default value is `0`, meaning no small file or zvol blocks are allocated in the special class.[S7]

The special device’s redundancy should match the pool’s normal-device redundancy because special vdevs store internal metadata and may also store small file blocks, and the OpenZFS documentation explicitly says the redundancy of a special device should match the redundancy of the other normal devices in the pool.[S1] This matters because a pool can become faulted when it has corrupted metadata or faulted devices with insufficient replicas to continue functioning, so a less-redundant special class can become the pool’s weaker failure point.[S1]
