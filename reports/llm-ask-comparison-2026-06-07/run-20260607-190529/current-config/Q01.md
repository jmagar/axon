  Assistant:
Bun does not run arbitrary dependency lifecycle scripts by default because those scripts can execute arbitrary code on your machine, which is a potential security risk and could allow malicious scripts to run during installation [S2][S3][S4].

To allow a specific dependency’s lifecycle scripts to run, the project must add that package name to the `trustedDependencies` array in its `package.json`, then install or re-install so Bun reads the field and runs that package’s lifecycle scripts [S2][S3].

```json
{
  "trustedDependencies": ["my-trusted-package"]
}
```
