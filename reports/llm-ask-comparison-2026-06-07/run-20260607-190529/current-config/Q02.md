PostgreSQL’s HOT optimization is possible when the update does not modify any columns referenced by the table’s indexes and there is enough free space on the page containing the old row to store the updated row version. [S1][S4]  

In current PostgreSQL documentation, summarizing indexes are excluded from the “indexed columns” restriction, and BRIN is identified as the only summarizing index method in the core distribution. [S4]  

When those conditions are met, HOT reduces two update overheads: it avoids creating new index entries for updated rows, and it allows obsolete row versions to be removed during normal operation, including `SELECT`s, instead of relying on periodic vacuum operations. [S1][S4]
