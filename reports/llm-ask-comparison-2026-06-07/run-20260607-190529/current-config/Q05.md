  Assistant:
Before adding devices to the special allocation class, the pool must already have at least one normal vdev, meaning a vdev that is not a `dedup` or `special` vdev. [S1]

Small-block placement is opt-in per dataset: add a special-class vdev to the pool, then set the dataset’s `special_small_blocks` property to a nonzero threshold size. [S1][S7] Blocks smaller than or equal to that threshold are assigned to the special allocation class, while larger blocks go to the regular class; the default value is `0`, which means no small file or zvol blocks are allocated in the special class. [S7]

The special device’s redundancy should match the pool’s normal-device redundancy because special vdevs store internal metadata and can also store small file blocks, and OpenZFS explicitly states that the redundancy of a special device should match the redundancy of the other normal devices in the pool. [S1] More generally, OpenZFS describes pools without sufficient redundancy as vulnerable to corruption or unavailable data, and a faulted pool can result from corrupted metadata or insufficient replicas to continue functioning. [S1]
