  Assistant:
The components included when hashing a KV-cache block are the parent hash value, the block's tokens (as a tuple), and extra hashes that ensure the block's uniqueness, which can include LoRA IDs, multi-modality input hashes, and cache salts for multi-tenant isolation [S1]. The system only caches full blocks [S1, S2]. Regarding hash collision risk, as of v0.11, the default hashing algorithm is `sha256`, which was implemented to address collision risks present in previous versions [S1, S2, S5]. Furthermore, for deterministic caching across different environments, using `sha256_cbor` with `cbor2` serialization is recommended [S1, S5].
