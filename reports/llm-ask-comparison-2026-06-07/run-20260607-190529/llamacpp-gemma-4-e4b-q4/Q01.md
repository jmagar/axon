By default, Bun does not execute arbitrary lifecycle scripts for installed dependencies because doing so poses a potential security risk, as these scripts could run arbitrary code on your machine [S2][S3][S15]. While Bun has automatically pre-verified a set of popular packages, running scripts requires explicit trust [S15]. To allow a specific dependency's lifecycle scripts to run, the project must add the package's name to the `trustedDependencies` array within the `package.json` file and then re-run the installation process [S2][S3]. Alternatively, you can use the `bun add --trust [package]` command to add the package and its transitive dependencies to this list [S16].
