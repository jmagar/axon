  Assistant:
Subnet routers and exit nodes are both methods of routing traffic within the Tailscale ecosystem, but they solve fundamentally different network problems [S18]. A **subnet router** is used to extend your secure Tailscale network (tailnet) to include and provide access to existing, non-Tailscale devices or entire private subnets, such as an office LAN or an AWS VPC, without installing the Tailscale client on every device [S1], [S11], [S12]. In contrast, an **exit node** routes *all* of a device's outbound internet traffic through itself, effectively making the exit node act as a VPN server, which is useful for accessing geo-restricted content or improving privacy when connecting from an untrusted network [S12], [S8].

Regarding the technical behavior, subnet routers are responsible for enabling tailnet devices to reach private subnets, whereas exit nodes manage the general internet egress path for the entire tailnet's traffic [S1], [S12]. When traffic flows from a device located behind a subnet router, Tailscale uses Source Network Address Translation (SNAT) by default, which means the traffic appears to originate from the subnet router itself rather than the actual source device [S1], [S5]. If preserving the original source IP address is necessary for the use case, SNAT can be disabled, though this requires manual configuration of return routes [S5].
