## Summary

A substantial fraction of the documentation-contract's "Final Docs Tree" leaf
files are still 15–25 line placeholder stubs, no real crate has a top-level
human `README.md`, generated markdown headers name the wrong generator command,
and one generated schema path is a placeholder while the real artifact lives
under a different filename. The contract's own freshness checks
(`cargo xtask docs generate --check`, `cargo xtask schemas generate --check`,
`cargo xtask check-doc-contracts`) are presumably catching some of this, but
the stubs and naming drift have persisted.

## Gap inventory

### 1. Stub-only reference docs (contract required, content missing)

All of these exist only as ~15–25 line H1-plus-bullets placeholders:

- `docs/reference/surfaces/{web,palette,android,chrome-extension,presentation}.md` — all 5 stubs
- `docs/reference/memory/{overview,decay,review}.md` — all 3 stubs
- `docs/reference/operations/{doctor,backup-restore,reset,troubleshooting}.md` — all 4 stubs
- `docs/reference/runtime/{jobs,observability,providers,storage,schema,auth,security,redaction,pruning}.md` — 9 of the runtime hand-docs are stubs (`ledger.md` and `memory.md` are substantive)
- `docs/reference/rest/{overview,routes}.md` — stubs
- `docs/reference/mcp/tool-contract.md` — stub
- `docs/reference/api/stage-results.md` — stub
- `docs/reference/config/examples.md` — stub
- `docs/reference/sources/{adding-source,url-normalization,metadata-payload,parsing,chunking,source-graph}.md` — stubs
- `docs/architecture/{repo-structure,crate-structure,source-pipeline,boundary-map,dependency-layering}.md` — stubs (`overview.md` is substantive)

### 2. Per-crate top-level README.md — none exist

The contract (and `docs/pipeline-unification/crates/README.md` lines 14–23)
targets every crate having a human `README.md` generated from or checked against
`crates/<name>/src/CLAUDE.md` plus rustdoc. Verified: no real crate
(`find crates -maxdepth 2 -name README.md`) has one. Only `src/CLAUDE.md`
exists per crate (with `AGENTS.md`/`GEMINI.md` symlinks), which satisfies the
agent-contract half but not the human-README half.

### 3. Generated-markdown header drift

Every generated `.md` under `docs/reference/{cli,rest,api,config,sources}/`
carries `<!-- generated by cargo xtask schemas ... -->`, but the contract's
Generated Artifacts table prescribes `cargo xtask docs …` for markdown outputs
(reserving `schemas …` for the JSON artifacts). The CLI markdown generator was
re-enabled in this session (it now emits a real grouped command table), but it
still labels itself `schemas cli` rather than `docs cli`.

### 4. Schema path/naming drift

- `docs/reference/runtime/database.schema.json` (contract line 90, dot) is a
  19-line placeholder; the real generated DB schema lives at the hyphenated
  `docs/reference/runtime/database-schema.json` (3583 lines). The contract
  path and the schemas-README path disagree; the live tree has both.
- `presentation/tokens.{md,schema.json}` live at `docs/reference/presentation/`
  but the contract names `docs/reference/surfaces/presentation/`.

### 5. MCP tool-schema doc not generator-produced

`docs/reference/mcp/tool-schema.md` is hand-authored with a "Source of Truth"
header (160 lines), but the contract requires it to be generated by
`cargo xtask docs mcp`. The sibling `tool-schema.json` (4083 lines) is the
real artifact.

## Suggested acceptance criteria

- Every path in `documentation-contract.md` "Final Docs Tree" is either
  substantive (>50 lines of real content) or removed from the contract.
- Every real crate has `crates/<name>/README.md` (generated from or checked
  against `src/CLAUDE.md`), or the contract's README goal is explicitly
  descoped.
- Generated `.md` headers match the contract's prescribed generator command,
  or the contract is updated to match the actual `schemas …` command.
- `database.schema.json` vs `database-schema.json` is reconciled to one path.
- `mcp/tool-schema.md` is either generator-produced or the contract's
  "generated" requirement for it is amended.
- The freshness checks (`check-doc-contracts`, `check-doc-links`) pass clean.
