# REST API v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Mount a full-parity REST API at `/api/v1/` on the existing axum server (port 49000), exposing all service functions behind the same `AXON_WEB_API_TOKEN` auth used by the WebSocket gate.

**Architecture:** Thin dispatcher — one handler file per domain group, all handlers call `crates::services::*` directly. A tower middleware layer validates the API token once at the router boundary. Async jobs use fire-and-forget by default; clients poll with `GET .../jobs/{id}`.

**Tech Stack:** axum 0.8, tower middleware, serde_json, `tower::ServiceExt::oneshot()` for tests. **Crate name: `axon`** (all `cargo` commands use `-p axon`).

**Route design note:** Job sub-resources use `/jobs/` nesting (e.g. `GET /api/v1/crawl/jobs`) to avoid conflicts between `POST /api/v1/crawl` (start job) and future resource-level operations. This is intentional and differs from flat-route alternatives.

## Background: Key Invariants

Read these before implementing any handler:

1. **SSRF guard** — call `crates::core::http::validate_url(url)?` before every user-supplied URL reaches a service. Return 400 on `Err`.
2. **`_raw` variants** — use `crawl_status_raw`, `embed_status_raw`, `extract_status_raw`, `ingest_status_raw` (return typed structs) not `crawl_status`, etc. (return CLI-formatted payloads). `refresh_status` has no `_raw` variant; use it directly and check `result.job.is_none()` for 404.
3. **`classify_target` import** — `use crate::crates::services::ingest::classify_target;` — it is re-exported from `crates/services/ingest.rs` via `pub use classify::classify_target`. The two-arg signature is `classify_target(target: &str, include_source: bool) -> Result<IngestSource, Box<dyn Error>>`. Pass `state.cfg.github_include_source` as the second arg.
4. **`ask_graph` field** — when the `POST /api/v1/ask` body includes `graph: bool`, clone `cfg` and set `cfg.ask_graph = body.graph`.
5. **List functions** — `crawl_list_raw`, `embed_list_raw`, `extract_list_raw`, `ingest_list_raw` take `limit: i64, offset: i64` directly (not `Pagination`). `refresh_list` also takes `limit: i64, offset: i64`.
6. **`system::sources` and `system::domains`** — both take `Pagination { limit: usize, offset: usize }` as the second argument, not a `bool`.
7. **`system::detailed_domains`** — separate function for `?detailed=true` domains queries. Signature: `detailed_domains(cfg: &Config) -> Result<DetailedDomainsResult, Box<dyn Error>>`.
8. **Monolith policy** — ≤500 lines/file, ≤120 lines/function.
9. **Handler naming** — all handler functions are named with a `_handler` suffix (e.g. `crawl_start_handler`). The `router()` function uses explicit alias blocks to expose them under clean names. See Task 1 for the alias pattern.
10. **`WsQuery`** — private struct in `crates/web.rs`, accessible as `super::WsQuery` from child module `api.rs`.
11. **`AppState`** — `pub(crate) struct AppState` in `crates/web.rs`, accessible as `super::AppState`.
12. **HTTP status for job start** — return `202 Accepted` for all fire-and-forget job creation (crawl, embed, extract, ingest, refresh, graph/build).
13. **`deny_unknown_fields` + axum** — axum returns `422 Unprocessable Entity` for serde deserialization failures. Tests should assert 422, not 400, for unknown-field rejections.

  async fn api_auth_middleware(
      State(state): State<Arc<AppState>>,
      Query(params): Query<WsQuery>,
      request: Request<Body>,
      next: Next,
  ) -> Response {
      match check_auth(request.headers(), params.token.as_deref(), state.api_token.as_deref()) {
          AuthOutcome::Token => next.run(request).await,
          AuthOutcome::Denied(_) => unauthorized(),
      }
  }

      (StatusCode::INTERNAL_SERVER_ERROR, axum::Json(json!({ "error": e.to_string() }))).into_response()

      (StatusCode::NOT_FOUND, axum::Json(json!({ "error": msg }))).into_response()
  }

  pub(crate) fn bad_request(msg: &str) -> Response {
      (StatusCode::BAD_REQUEST, axum::Json(json!({ "error": msg }))).into_response()
  }

  pub(crate) fn unauthorized() -> Response {
      (StatusCode::UNAUTHORIZED, axum::Json(json!({ "error": "unauthorized" }))).into_response()
  }

  use axum::{
      Json,

      if body.urls.is_empty() {
          return bad_request("urls must not be empty");

      for url in &body.urls {

              return bad_request(&format!("invalid URL {url}: {e}"));

          return bad_request("invalid UUID");

          Ok(Some(job)) => Json(job).into_response(),
          Ok(None) => not_found("crawl job not found"),
          Err(e) => service_err(e),
      }
  }

          Ok(canceled) => Json(serde_json::json!({ "canceled": canceled })).into_response(),
          Err(e) => service_err(e),
      }

## Gotchas for Implementers

1. **Crate name is `axon`** — all `cargo` commands use `-p axon`, not `-p axon_cli`. Running `-p axon_cli` will error with "package ID specification did not match any packages."

2. **Handler name suffix** — all handler functions end in `_handler` (e.g. `crawl_start_handler`). The `router()` function in `api.rs` uses explicit `use module::{func_handler as func}` aliases to expose clean names to the route declarations.

3. **`system::sources` and `system::domains` both take `Pagination`** — not a `bool`. The `?detailed=true` path on `/api/v1/domains` routes to `system::detailed_domains(cfg)` instead.
