## User Request

Normalize `axon_rust` to a standard single-crate Rust layout — rename `crates/` to `src/`, move `lib.rs`/`main.rs` into `src/`, and eliminate the non-standard `Cargo.toml` path overrides.

## Session Overview

Concluded the project is a single-crate Rust project masquerading as a workspace. Executed the rename in an isolated git worktree, fixed 1112 import paths via mass sed, resolved multiple CI gate failures, ran code review and PR comment workflows, merged PR #68, then fixed two pre-existing test failures that surfaced due to a docker-compose path relocation.

## Key Findings

- `Cargo.toml` `[lib] path = "lib.rs"` and `[[bin]] path = "main.rs"` were the sole reason source files lived at repo root — removing them is safe and Cargo auto-discovers `src/lib.rs` + `src/main.rs`
- `crates/` was not a workspace member directory — it was a plain Rust module tree; `crates.rs` was a trivial re-export shim (`pub mod cli; pub mod core; ...`)
- 1112 occurrences of `crate::crates::` needed replacing across the codebase
- `sqlx::migrate!` path is relative to `CARGO_MANIFEST_DIR` (repo root) at compile time — `src/jobs/lite/migrations` is correct (`src/jobs/lite/store.rs:55`)
- `xtask/src/checks/mcp_http.rs` had hardcoded `crates/` file paths used as both runtime paths and error message strings — both needed updating
- `.monolith-allowlist` had `crates/`-prefixed paths that the monolith checker couldn't match after rename
- `Cargo.lock` must be regenerated after any `Cargo.toml` version bump or it fails `--locked` CI gates
- `docs/MCP-TOOL-SCHEMA.md` is generated by `scripts/generate_mcp_schema_doc.py` — must be regenerated when schema file path changes
- 5 scripts (`generate_mcp_schema_doc.py`, `enforce_no_legacy_symbols.py`, `qdrant_quality_runtime.py`, `qdrant_quality_settings.py`, `mcp_doc_renderer.py`) had hardcoded `crates/` paths
- CI `mcp-smoke` and `compose_env_contract` failures were pre-existing but masked — caused by `config/docker-compose.services.yaml` not existing; root compose is now `docker-compose.yaml`

## Technical Decisions

- **Worktree isolation**: Used `EnterWorktree` to keep `main` clean during the refactor; all changes made on `worktree-src-layout` branch
- **Mass sed over manual edits**: 1112 occurrences made manual editing impractical; `find src -name "*.rs" | xargs sed -i` was safe because the pattern `crate::crates::` is unambiguous
- **`crates.rs` inlined into `src/lib.rs`**: Eliminated the indirection layer entirely rather than keeping it as `src/crates.rs`; cleaner and follows standard layout
- **Squash merge**: Single logical change warranted squash to keep main history clean despite multiple fixup commits during CI iteration
- **`docker-compose.services.yaml` → `docker-compose.yaml`**: User confirmed compose had already been relocated; updated tests and CI to match rather than restoring the old path

- **Use `src/crates.rs` shim**: Instead inlined declarations directly into `src/lib.rs`, eliminating one indirection level.
