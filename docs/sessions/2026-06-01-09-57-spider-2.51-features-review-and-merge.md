## Key Findings

- **P1 etag bug was real, not "irreducible":** `reconcile_targets` (`src/crawl/engine/etag.rs:196`) used `seeded ∧ ¬arrived`, which cannot distinguish a genuine 304 skip from a no-longer-discovered page → re-appends deleted pages as `changed=false` zombies. Spider's 304 short-circuit (`website.rs:7375`, `return Default::default()`) runs **inside** the per-URL fetch task, so a 304-skipped URL is in `Website::get_links()` while an undiscovered URL is not. Fix: gate on `... ∧ visited`.
- **AIMD is dead code in spider 2.51.197:** `AIMDController` exists but no public `Website`/`configuration` method attaches it; only `auto_throttle` (latency-based delay) is wireable, which does not match the 429/403 backoff intent. Documented, not faked.
- **`--etag-conditional` is independent of `--cache`:** no code path couples them; the "Requires --cache" doc claim was false (reconciliation relinks from Axon's own `markdown.old`).
- **`docs/SPIDER-FEATURE-FLAGS.md` pre-existing drift:** listed `firewall` as enabled (it is disabled in `Cargo.toml`, panics in CI) and miscounted flags.
- **Parallel version collisions:** PR #152 shipped 4.17.0 and PR #151 shipped 4.18.0 while this branch was open, forcing two rebump/relabel steps and leaving a 4.18.0 duplicate on main (bead `ptug`).

## Technical Decisions

- **Handled the 11 review threads directly** rather than re-dispatching the sub-agent, because the P1 challenged the agent's own "zombie-safe by construction" claim and needed fresh-eyes design.
- **Visited-set gate** chosen over documenting-as-limitation: it makes the etag feature correct with a public API (`get_links()`) instead of accepting stale content.
- **SSRF redirect guard: confirm-only, zero code change** — kept Axon's 3-layer defense; removing security code requires demonstrated equivalence, not inference.
- **Flaky stack overflow fixed at root** by running the two live-`crawl_raw()` etag tests on a 16 MB-stack thread (`block_on_big_stack`), instead of touching CI/just env — self-contained to the test file and works in local + CI nextest.
- **Version bumped to 4.18.0** at merge (not 4.17.1) since these are features and #152 had taken 4.17.0.
