## Key Findings

- **Unused imports** in `metrics.rs:4,6`, `collector.rs:7,10`, `map.rs:4,14`, `files.rs:10,11,19` — all re-exports left over from monolith splits that were removed from callers but not from the re-export files.
- **`hickory-net/proto/resolver 0.26.0`** had two CVEs published 2026-05-01 (RUSTSEC-2026-0119: CPU exhaustion O(n²) name compression; RUSTSEC-2026-0120: NSEC3 unbounded loop). Fix: `cargo update` to 0.26.1, no spider bump needed.
- **`mcp-transport-modes`** check grep for `AXON_MCP_TRANSPORT` in `build_config.rs` — the string was only in `helpers.rs` (the actual resolver). Fix was an anchor comment in `build_config.rs`.
- **`mcp-oauth-smoke` stdio crash**: `axon mcp` without `--transport http` starts both stdio and HTTP transports. In CI, the background process's stdin is closed, causing `ConnectionClosed("initialize request")` on the stdio side, crashing the whole server before the HTTP port is bound.
- **`web-lint-test`** referenced `apps/web/pnpm-lock.yaml` which doesn't exist (webapp removed). Pre-existing failure on main.
- **`no-next-middleware`** referenced `scripts/check_no_next_middleware.sh` which was deleted with the webapp.
- **`msrv` job** is NOT a no-op — `rust-version = "1.94.0"` in `Cargo.toml` means 1.94.0 is the declared MSRV; the job correctly validates against it.

## Technical Decisions

- **Dummy TEI/Qdrant URLs in `mcp-oauth-smoke`**: The test only exercises the auth layer (`/mcp` returns 401). TEI and Qdrant are never called. Supplying `http://127.0.0.1:52000` and `http://127.0.0.1:53333` satisfies `into_config()` validation without needing running services.
- **`--transport http` in both smoke tests**: Cleaner than redirecting stdin from `/dev/null` — eliminates the stdio transport entirely rather than masking the EOF.
- **Docker image cache key uses manual version suffix** (`tei-cpu-v1`, `qdrant-1.13.1`): `cpu-latest` is not a pinned tag, so cache invalidation is done by bumping the `v1` suffix rather than relying on the tag changing.
- **Removed bare `docker compose down` from teardown**: No `docker-compose.yaml` exists at the repo root; the `|| true` was masking the silent failure. Only the explicit `-f config/docker-compose.services.yaml` teardown is needed.
