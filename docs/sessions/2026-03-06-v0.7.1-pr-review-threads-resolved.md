## Key Findings

- `EnvGuard::set()` in `tei/tests.rs` used `unsafe { env::set_var }` with SAFETY comment claiming "single-threaded" — tokio tests run in parallel across threads; `serial_test = "3"` was already in dev-deps, only needed `#[serial]` annotation on the one non-serialized test
- `isBackgroundRef.current` was left `true` on background command error in `CmdKPalette.tsx:200-213` — next Cmd+K open would spuriously enter 'running' with no active command
- `map_map_payload()` in `crates/services/map.rs` was a one-line trivial wrapper (`Ok(MapResult { payload })`) called once; two test files imported it and needed updating after removal
- `tabs` in the `?doc=` `useEffect` deps at `editor/page.tsx:88` caused `controller.abort()` to fire whenever any unrelated tab state changed; fix: local `tabsRef` ref tracks tabs without being a dependency
- `session.filename` (not `session.id`) is the Claude session UUID required by `resumeWorkspaceSession`; `session.id` is a SHA256 route hash — using it would silently break session resume

## Technical Decisions

- **`tabsRef` in `editor/page.tsx`** rather than modifying `use-tabs.ts` — `use-tabs.ts` has an internal `tabsRef` but doesn't expose it; adding a local ref in the page is the minimal, non-breaking fix
- **Won't-fix for sessionId thread** — cubic-dev-ai suggestion (`session.id`) was factually wrong; verified by tracing through `use-ws-messages.ts` which passes the value directly as a Claude session ID
- **Inline rather than rename** for `map_map_payload` — the function existed solely to construct `MapResult { payload }`; inlining removes indirection with no behavior change
- **`#[serial]` annotation only** (not `#[serial]` on all env tests) — only `tei_embed_retries_on_500` sets env vars in the general test suite; other env-guarded tests already had `#[serial]`
